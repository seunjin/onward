# 01. Product Strategy

## 1. 제품 한 줄 정의

Onward는 사용자의 해야 할 일, 습관, 스스로에게 건 도전을 기록하고, 매일의 실천이 어떻게 쌓이고 있는지 보여주는 실행 기록 서비스다.

## 2. 제품이 해결하려는 문제

기존의 할 일 앱은 완료 체크에는 강하지만, "얼마나 꾸준히 해왔는가"를 잘 보여주지 못한다.
습관 앱은 반복 실천에는 강하지만, 개인 목표나 짧은 도전 과제를 함께 관리하기 어렵다.
Onward는 이 둘을 분리하지 않고 하나의 실행 기록으로 묶는다.

핵심 문제는 아래와 같다.

- 오늘 무엇을 해야 하는지와 장기적으로 무엇을 쌓고 있는지가 분리되어 있다.
- 하루 이틀 놓치면 실패한 느낌이 커져서 다시 시작하기 어렵다.
- 사용자는 완벽한 수행보다, 멈추지 않고 이어가는 감각을 원한다.

## 3. 핵심 가치 제안

### Progress over perfection

Onward는 완벽한 성공률보다 누적된 실천과 재시작 가능성을 강조한다.

### 기록이 남는 실행

체크 한 번으로 끝나는 것이 아니라, 사용자는 일별 기록과 주간/월간 그래프를 통해 자신의 흐름을 본다.

### 같은 도구로 다른 목표를 관리

사용자는 할 일, 습관, 도전을 각각 다른 앱에 나눠두지 않고 하나의 구조 안에서 관리한다.

## 4. 핵심 사용자

### 1차 타깃

- 자기관리 의지가 있지만 완벽주의 때문에 중간에 자주 멈추는 사용자
- 할 일 앱과 습관 앱을 동시에 쓰다가 관리 피로를 느끼는 사용자
- 공부, 운동, 독서, 사이드 프로젝트 같은 개인 루틴과 도전을 함께 관리하려는 사용자

### 2차 타깃

- 30일 챌린지, 프로젝트 루틴, 개인 성장 목표를 시각적으로 추적하고 싶은 사용자
- 회고를 간단하게 남기고 싶은 사용자

## 5. 핵심 제품 원칙

### 1. 오늘의 행동이 가장 먼저 보여야 한다

앱/웹 첫 화면은 통계보다 오늘 해야 할 항목과 오늘의 기록 상태를 우선한다.

### 2. 실패보다 누적을 강조한다

연속 기록이 깨졌더라도 전체 누적, 최근 7일 흐름, 재시작 시점을 자연스럽게 보여준다.

### 3. 입력 비용은 낮고, 피드백 가치는 높아야 한다

하루 기록은 1~2번 탭으로 끝나야 하고, 그 결과는 그래프와 요약으로 되돌아와야 한다.

### 4. 웹과 앱의 역할은 같지 않아도 된다

앱은 빠른 체크인과 알림 대응에 강해야 한다.
웹은 긴 입력, 편집, 통계 탐색에 강해야 한다.

### 5. 서비스 웹과 마케팅 사이트는 분리할 수 있어야 한다

로그인 이후의 서비스 웹은 빠른 상호작용이 중요하다.
SEO가 필요한 랜딩 페이지는 이후 별도 앱으로 분리해 운영할 수 있어야 한다.

## 6. 정보 구조

초기 제품은 아래 네 개의 축으로 구성한다.

### Dashboard

- 오늘 해야 할 항목 목록
- 오늘 완료/실행 상태
- 최근 기록 요약
- 현재 streak, 최근 7일 요약

### Goals

- 해야 할 일
- 습관
- 도전
- 생성, 수정, 보관

### Records

- 하루 단위 실행 기록
- 완료 여부 또는 수치 입력
- 간단한 메모

### Insights

- 일/주/월 그래프
- 누적 실행 횟수
- 성공률
- 현재 streak / 최고 streak

## 7. MVP 범위

## 반드시 포함

- 이메일 또는 소셜 로그인 중 1개 방식
- Goal 생성/수정/보관
- Goal 유형 구분: `todo`, `habit`, `challenge`
- 반복 규칙 설정: 매일, 특정 요일, 기간형 도전
- 일별 기록 생성/수정
- 오늘 화면
- 주간/월간 그래프
- streak 및 누적 통계
- 웹과 앱 모두에서 핵심 흐름 제공

## 있으면 좋지만 MVP에서 제외 가능

- 친구/소셜 기능
- 공개 프로필
- 고급 리포트
- AI 추천
- 팀 챌린지
- 위젯
- 캘린더 외부 연동

## 8. 핵심 사용자 플로우

### 플로우 A: 첫 사용

1. 회원가입
2. 첫 Goal 생성
3. 반복 규칙 선택
4. 오늘 화면 진입
5. 첫 기록 체크
6. 즉시 그래프/누적값 확인

### 플로우 B: 매일 체크인

1. 오늘 화면 진입
2. 오늘 해야 할 항목 확인
3. 완료 또는 진행 기록 입력
4. streak/누적 상태 갱신 확인

### 플로우 C: 주간 회고

1. Insights 화면 진입
2. 최근 7일/30일 그래프 확인
3. 어떤 목표가 유지되었는지 확인
4. 필요한 Goal 수정 또는 도전 재설정

## 9. 화면 우선순위

### 웹

- 로그인
- 대시보드
- Goal 목록/상세/편집
- 기록 캘린더/그래프
- 설정

초기 웹은 로그인 이후 서비스 사용 흐름에 집중한다.
퍼블릭 랜딩 페이지는 MVP 범위에서 제외하고 추후 별도 앱으로 추가한다.

### 모바일

- 로그인
- 오늘 화면
- Goal 생성/편집
- 간단 통계
- 알림 진입 기반 빠른 기록

## 10. 성공 지표

MVP 단계에서는 아래를 가장 중요한 지표로 본다.

- 가입 후 첫 Goal 생성 전환율
- 첫 기록 생성 전환율
- 7일 내 재방문율
- 주간 활성 사용자 대비 기록 생성 비율
- Goal 생성 후 7일 내 3회 이상 기록한 사용자 비율

랜딩 전환율 같은 마케팅 지표는 초기 핵심 지표에서 제외한다.

## 11. 초기 릴리즈 기준

초기 릴리즈는 아래 조건을 만족하면 가능하다.

- 웹과 앱 모두에서 같은 계정으로 로그인 가능
- Goal 생성 후 오늘 화면에 노출됨
- 기록 체크가 서버에 저장되고 통계에 반영됨
- 최근 7일 그래프가 정상 동작함
- streak 계산이 기본 시나리오에서 일관되게 동작함
- 서비스 웹은 SPA 기준으로 부드럽게 탐색 가능해야 함
