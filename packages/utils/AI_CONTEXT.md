# Utils 패키지

## 범위

- 특정 앱이나 제품 모듈에 속하지 않는 범용 헬퍼를 담당한다.

## 주의사항

- 헬퍼는 작고 일반적인 성격으로 유지한다.
- 제품 특화 로직은 `@onward/domain`으로 보낸다.
- 워크스페이스에서 안전하게 공유 가능한 헬퍼는 중복 작성하지 않는다.
- 앱 전용 편의 함수를 범용 유틸처럼 포장해서 넣지 않는다.

## 작업 규칙

- 날짜, 숫자, 문자열 같은 범용 helper만 둔다.
- 다른 패키지에서도 이름만 보고 역할이 드러나는 함수명을 사용한다.
- 제품 맥락이 생기기 시작하면 `domain`으로 이동할지 먼저 판단한다.

## 변경 후 확인

- 최소 `pnpm --filter @onward/utils typecheck`를 실행한다.
