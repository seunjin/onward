# Contracts 패키지

## 범위

- 공용 제품 상수, enum, 앱/서버 간 공유 타입을 담당한다.

## 주의사항

- 이 패키지는 프레임워크에 의존하지 않게 유지한다.
- 도메인 동작 로직까지 이 패키지에 넣지 않는다.
- 특정 화면에서만 잠깐 쓰는 임시 타입을 이 패키지에 올리지 않는다.

## 작업 규칙

- 앱과 API가 같은 제품 용어를 써야 할 때 이곳에 추가한다.
- enum, status, DTO shape, 공용 상수처럼 계약 자체를 표현하는 내용만 둔다.
- 새 타입을 추가할 때는 앱과 서버 양쪽 사용처를 기준으로 이름을 정한다.

## 변경 후 확인

- 최소 `pnpm --filter @onward/contracts typecheck`를 실행한다.
- 빌드 산출을 쓰는 패키지이므로 필요 시 `pnpm --filter @onward/contracts build`도 확인한다.
